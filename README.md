# 杏铃同步台

本项目是一个现代化、可视化的 Telegram 频道同步与历史消息搬运工具。它采用前后端分离的 Web UI 面板进行管理，支持频道实时监听同步、断点续传，并首创了 **Aiogram + Pyrofork 双引擎架构**，以满足不同场景下海量历史数据的安全迁移。

## ✨ 核心特性

  - 🚀 **现代化 Web 控制台**：基于 Vue3 + Tailwind CSS 构建，提供极佳的交互体验和实时进度/日志展示。
  - 🔄 **毫秒级实时同步**：24小时监听源频道的新消息和修改操作，完美 1:1 搬运（包含文本格式、媒体文件）。
  - ⏸ **无感断点续传**：内置 SQLite 数据库精准记录每一条消息的映射关系。任务随时暂停/重启，**绝不漏发、绝不重发**。
  - 🛡️ **高级正则过滤**：支持对文本、媒体文件名的正则表达式匹配。可在同步时**自动替换敏感词、删除广告文本**，或**直接跳过特定的图片/视频文件**。
  - 📦 **媒体组打包同步**：智能识别 Telegram 相册（Media Group），完美还原原频道媒体组排版，绝不拆散发送。
  - 🔍 **智能标识解析**：支持直接粘贴频道**公开链接**、**@用户名**或数字 ID，程序会自动解析并建立同步路径。
  - ⚙️ **智能双引擎架构 (按需加载)**：
      - **Aiogram 实时引擎**：负责高效率、高稳定性的 24 小时实时监听同步。
      - **Pyrofork 历史引擎**：配置 API 账号后自动激活，专门负责秒级拉取大规模历史列表。
  - 📦 **全能历史数据同步**：
      - **① API 拉取**：利用辅助账号全自动高速遍历历史消息。
      - **② JSON 导入 (强烈推荐)**：读取官方导出的本地 JSON 历史记录与媒体文件，安全、稳定、极速。
      - **③ ID 盲猜**：兜底方案，通过 ID 暴力遍历。**内置连续失败熔断机制**，保护 API 账号安全。

## 🛠️ 安装与部署

### 1\. 环境要求

  - **Python 3.9 \~ 3.13** \> ⚠️ **避坑警告**：请**不要**使用 Python 3.14 或更新的测试版本！本项目底层加解密依赖 `tgcrypto`（包含 C 语言扩展），新版 Python 暂无预编译程序，会导致安装或启动报错。
  - 建议在独立的虚拟环境 (`venv`) 中运行项目。

### 2\. 获取代码与依赖

```bash
git clone https://github.com/RRHTY/tg-channel-sync.git
cd tg-channel-sync
pip install -r requirements.txt
```

### 3\. 配置环境变量 (最重要)

[cite_start]将项目根目录下的 `.env.example` 复制并重命名为 `.env` [cite: 3]。

```bash
cp .env.example .env
```

打开 `.env` 文件并填入你的配置信息：

```ini
# ================= Telegram API 配置 (可选) =================
# 填写后可解锁 API 拉取与 ID 盲猜模式
API_ID=12345678
API_HASH=你的_api_hash_字符串

# ================= Bot 配置 (必填) =================
# 从 @BotFather 申请获取
BOT_TOKEN=123456789:ABCDefghIJKLmnop...

# ================= 系统运行配置 =================
# Web 服务运行端口 (默认 8011)
PORT=8011

# 盲猜模式连续失败熔断阈值 (默认 10)
MAX_FAILS=10
```

> 💡 **提示**：如果你只想使用 **实时同步** 和 **JSON 本地导入**，可以让 `API_ID` 保持为空，系统将自动以纯 Bot 模式运行。

### 4\. 启动项目

```bash
python main.py
```

启动成功后，在浏览器中打开：`http://localhost:8011`（或您在 .env 中设置的端口）即可进入管理后台。

## 📖 使用指南

### 准备工作

请务必确保你的机器人（Bot）已经被添加为 **源频道** 和 **目标频道** 的管理员，并且拥有发布消息的权限！

### 内容过滤 (正则规则)

1.  在 Web 面板的“高级内容过滤”模块中添加规则。
2.  **文本匹配**：匹配到的文本会被替换为您设定的内容（留空则直接删除该段文字）。
3.  **媒体匹配**：若媒体的消息文本或文件名匹配正则，则该条消息/整个媒体组将被跳过。

### 模式 1：JSON 本地导入 (最安全稳定)

1.  在电脑端 Telegram，进入源频道 -\> 右上角菜单 -\> `Export channel history`（导出频道历史）。
2.  勾选图片/视频等媒体选项，导出格式选择 `Machine-readable JSON`。
3.  在 Web 面板的“JSON 导入”选项卡中，填入生成的 `result.json` 的 **本地绝对路径**（例如 `D:\TG_Export\result.json`）。
4.  设定好延时，点击启动。机器人会自动读取路径下的媒体文件并根据过滤规则进行上传。

### 模式 2：API 官方拉取 (全自动)

1.  确保 `.env` 中已填写正确的 `API_ID`。
2.  首次启动时，**请留意终端 (控制台)**，系统会提示您输入辅助账号的手机号码和 TG 验证码。
3.  在 Web 面板选择“API 拉取”，输入起始消息 ID 或让其自动计算范围，点击启动。

## ❓ 常见问题 (FAQ)

**Q: 点击“停止任务”后，为什么按钮会变成橙色进度条？**
**A**: 为了保证数据库记录的绝对安全，程序在收到停止指令后，会等待当前正在执行的休眠（Delay）和网络请求彻底完毕。橙色进度条是安全退出的缓冲时间。

**Q: 遇到 `port 8011 is already in use` 怎么办？**
**A**: 说明 8011 端口被占用。请在 `.env` 中修改 `PORT` 变量为一个新端口（如 8012），然后重启程序。

**Q: 为什么媒体组会被重复发送？**
**A**: 本项目已针对底层库 `Pyrofork` 存在的解析 Bug 进行了特殊规避处理。请确保您使用的是最新版本的代码，系统会自动识别“假性失败”并防止重复发送。
