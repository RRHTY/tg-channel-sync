# tg-channel-sync
本项目是一个现代化、可视化的 Telegram 频道同步与历史消息搬运工具。它采用前后端分离的 Web UI 面板进行管理，支持频道实时监听同步、断点续传，并首创了 **Aiogram + Pyrofork 双引擎架构**，以满足不同场景下海量历史数据的安全迁移。

## ✨ 核心特性

  - 🚀 **现代化 Web 控制台**：基于 Vue3 + Tailwind CSS 构建，提供极佳的交互体验和实时进度/日志展示。
  - 🔄 **毫秒级实时同步**：24小时监听源频道的新消息和修改操作，完美 1:1 搬运（包含文本格式、媒体文件）。
  - ⏸ **无感断点续传**：内置 SQLite 数据库精准记录每一条消息的映射关系。任务随时暂停/重启，**绝不漏发、绝不重发**。
  - 正则匹配！随心控制文本
  - ⚙️ **智能双引擎架构 (按需加载)**：
      - **纯 Bot 模式 (默认)**：无需申请 API ID，零封号风险，极速部署。
      - **API 用户辅助模式**：配置 API ID 后自动解锁，借用真实用户身份秒级拉取历史列表。
  - 📦 **全能历史数据同步**：
      - **① API 拉取**：全自动高速遍历历史消息。
      - **② JSON 导入 (强烈推荐)**：读取官方导出的本地 JSON 历史记录与媒体文件，安全、稳定、极速。
      - **③ ID 盲猜**：即使没有导出数据，且未配置 API，也能通过 ID 暴力遍历的终极兜底方案。

## 🛠️ 安装与部署

### 1\. 环境要求

  - **Python 3.9 \~ 3.13** \> ⚠️ **避坑警告**：请**不要**使用 Python 3.14 或更新的测试版本！本项目底层加解密依赖 `tgcrypto`（包含 C 语言扩展），新版 Python 暂无预编译程序，会导致安装或启动报错。
  - 建议在独立的虚拟环境 (`venv`) 中运行项目

### 2\. 获取代码与依赖

```bash
git clone https://github.com/RRHTY/tg-channel-sync.git
cd tg-channel-sync
pip install -r requirements.txt
```

### 3\. 配置环境变量 (最重要)

将项目根目录下的 `.env.example` 复制并重命名为 `.env`。

```bash
cp .env.example .env
```

打开 `.env` 文件并填入你的配置信息：

```ini
# ==========================================
# Telegram API 配置 (非必填，填入以解锁 API 历史拉取模式)
# 申请地址: https://my.telegram.org
API_ID=12345678
API_HASH=你的_api_hash_字符串

# Bot Token (必填！)
# 从 @BotFather 申请获取
BOT_TOKEN=123456789:ABCDefghIJKLmnop...
# ==========================================
```

> 💡 **提示**：如果你只想使用 **实时同步**、**JSON 本地导入** 和 **ID盲猜**，你可以让 `API_ID` 和 `API_HASH` 保持为空，系统将自动以纯 Bot 模式运行。

### 4\. 启动项目

```bash
python main.py
```

启动成功后，在浏览器中打开：`http://localhost:8011` 或 `http://127.0.0.1:8011` 即可进入管理后台。

## 📖 使用指南

### 准备工作

请务必确保你的机器人（Bot）已经被添加为 **源频道** 和 **目标频道** 的管理员，并且拥有发布消息的权限！

### 模式 1：JSON 本地导入 (最安全稳定)

1.  在电脑端 Telegram，进入源频道 -\> 右上角菜单 -\> `Export channel history`（导出频道历史）。
2.  勾选图片/视频等媒体选项，导出格式选择 `Machine-readable JSON`。
3.  在 Web 面板的“JSON 导入”选项卡中，填入生成的 `result.json` 的 **本地绝对路径**（例如 `D:\TG_Export\result.json`）。
4.  设定好延时，点击启动。机器人会自动读取旁边的媒体文件夹并进行 1:1 上传。

### 模式 2：API 官方拉取 (全自动)

1.  确保 `.env` 中已填写正确的 `API_ID`。
2.  首次启动时，**请留意终端 (控制台)**，系统会提示你输入用来拉取数据的手机号码和 TG 验证码（验证成功后会生成本地 session 文件，后续无需再验证）。
3.  在 Web 面板选择“API 拉取”，输入起始 ID，点击启动即可解放双手。

### 模式 3：ID 盲猜法 (兜底方案)

纯 Bot 模式下即可运行。通过按数字顺序依次猜测消息 ID 进行复制转发。
**⚠️ 警告**：由于会产生大量 404 无效请求，非常容易触发 TG 官方的风控限制（FloodWait）。请将单条延时设置在 **3秒以上**，并做好了随时被官方熔断暂停的准备。

## ❓ 常见问题 (FAQ)

**Q: 安装依赖时报错 `Failed to build tgcrypto` 怎么办？**
**A**: 这是因为你的 Python 版本过高（例如 3.14+），官方尚未提供该版本的底层 C 环境预编译包。请将 Python 降级至 3.11 - 3.13 之间的稳定版本后重新安装。

**Q: 启动后控制台报错 `port 8011 is already in use` 怎么办？**
**A**: 这意味着 8011 端口被占用了（可能是上一次非正常退出导致）。你可以通过杀掉占用该端口的进程，或者在 `main.py` 最后一行将端口改为 `8011` 等其他未占用端口。

**Q: 点击“停止任务”后，为什么按钮会变成橙色进度条？**
**A**: 为了保证数据库中断点续传记录的绝对安全，程序在收到停止指令后，会等待当前正在执行的休眠(Delay)和网络请求彻底完毕。橙色进度条是安全退出的缓冲时间（约 3-5 秒）。

**Q: 为什么源频道发了新消息，机器人没有同步？**
**A**: 请检查：1. Web 面板左侧是否正确添加了对应的频道映射规则；2. 机器人是否是源频道的管理员；3. 源频道 ID 是否带上了 `-100` 前缀。
